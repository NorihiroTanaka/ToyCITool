# TASK-006: 将来的な拡張（Webダッシュボードとアーティファクト管理）

## 1. タスク概要
CUIベースの現在のツールを、GUI（Webブラウザ）で管理・閲覧できるようにし、ビルド生成物（アーティファクト）を保存・ダウンロードできるようにする。

## 2. 現状の課題
*   **現状:** ログはファイルシステム上のテキストファイル、操作はAPIまたはコマンド経由のみ。
*   **問題点:** 履歴の閲覧性が悪い。非エンジニアや、サーバーにSSHできないユーザーが状況を確認できない。ビルド成果物（exeやapkなど）の取り出しが面倒。

## 3. 仕様・要件（構想）

### Webダッシュボード
*   **技術スタック:**
    *   Backend: FastAPI (既存の `src/api.py` を拡張) + Jinja2 Templates または React/Vue.js (SPA)。
    *   Database: SQLite (現状のインメモリ/ファイル管理から移行) でジョブ履歴を永続化。
*   **機能:**
    *   ジョブ一覧ページ（ステータス、開始時間、所要時間）。
    *   ジョブ詳細ページ（リアルタイムログ表示、メタデータ表示）。
    *   手動実行ボタン。

### アーティファクト管理
*   **機能:**
    *   ジョブ定義で `artifacts: ["dist/*.zip"]` のようにパスを指定。
    *   ジョブ成功時、指定されたファイルを `artifacts/{job_id}/` にコピー・保存。
    *   Webダッシュボードからダウンロード可能にする。

## 4. 優先度
**Low**
（現状の「軽量CIツール」というコンセプトからは外れるため、フェーズ2以降の長期目標とする）
