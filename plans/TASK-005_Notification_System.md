# TASK-005: 通知機能の追加

## 1. タスク概要
ジョブの成功・失敗、開始・終了などのイベントを、外部システム（Discord, Slack, Emailなど）に通知する機能を実装する。

## 2. 現状の課題
*   **現状:** コンソールを見るか、ログファイルを確認しないと結果がわからない。
*   **問題点:** ビルド失敗に気づくのが遅れる。自動化のメリット（迅速なフィードバック）が薄れる。

## 3. 仕様・要件

### コアコンセプトへの適合
*   **Docker非依存 (No Docker Dependency):**
    *   通知エージェントなどを別コンテナで立ち上げる必要はなく、メインプロセス内から直接HTTPリクエスト等を送信するシンプルな実装とする。
*   **容易な設定 (Easy Configuration):**
    *   最低限の設定（例: `webhook_url` のみ）で動作するようにする。
    *   複雑なテンプレート設定などを強制せず、デフォルトで視認性の高い通知フォーマットを提供する。
    *   通知機能自体をオプショナルとし、設定が空であれば何も行わない（エラーにならない）挙動とする。

### 実装内容
*   **対象ファイル:**
    *   `src/core/notifier.py` (新規作成): 通知ロジックの抽象化。
    *   `src/core/job_service.py`: ジョブ完了時に通知メソッドを呼び出す。
*   **設定ファイル:** `config.yaml` にWebhook URLなどを設定。

### 期待される挙動
1.  ジョブ終了時（成功/失敗問わず、あるいは失敗時のみなど設定可能に）、通知処理をトリガーする。
2.  設定された通知チャンネル（今回はまず **Discord Webhook** を想定）にメッセージを送信する。
3.  メッセージには以下を含める：
    *   ジョブステータス（✅ Success / ❌ Failure）
    *   プロジェクト名 / ブランチ名
    *   コミットメッセージ（取得できていれば）
    *   ログへのリンク（将来的にWeb UIがあれば）、またはログの最後の数行。

### 拡張性
*   `Notifier` クラスを基底クラスとし、`DiscordNotifier`, `SlackNotifier` などのサブクラスを作成できる設計にする。

## 4. 優先度
**Low / Medium**
（機能的には必須ではないが、UX向上のために重要。Webhook URL一つで実装できるDiscord通知などはコスト対効果が高い）
